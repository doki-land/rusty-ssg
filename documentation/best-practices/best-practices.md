# 最佳实践

本指南提供了使用 Rusty SSG 构建静态站点的最佳实践，帮助您创建高性能、安全和可维护的网站。

## 项目结构

### 推荐的项目结构

为了保持项目的可维护性，建议采用以下项目结构：

```
my-site/
├── src/                # 源代码
│   ├── components/     # 组件（对于支持组件的编译器）
│   ├── layouts/        # 布局模板
│   ├── pages/          # 页面文件
│   └── styles/         # 样式文件
├── content/            # 内容文件（对于 Hugo 等编译器）
├── static/             # 静态资源
├── public/             # 构建输出目录
├── config.toml         # 配置文件
└── README.md           # 项目说明
```

### 命名约定

- **文件命名**：使用小写字母和连字符（如 `my-page.md`）
- **目录命名**：使用小写字母和连字符（如 `blog-posts`）
- **组件命名**：使用 PascalCase（如 `HeaderComponent.astro`）
- **变量命名**：使用 snake_case（如 `site_title`）

## 性能优化

### 构建性能

1. **增量构建**：使用增量构建减少构建时间
   ```bash
   # 对于支持增量构建的编译器
   <compiler-name> build --incremental
   ```

2. **并行处理**：利用 Rust 的并行处理能力
   - 确保编译器配置为使用多核
   - 对于大型站点，考虑增加构建机器的 CPU 核心数

3. **缓存策略**：
   - 使用编译器的缓存机制
   - 对于 CI/CD 环境，缓存 `target` 目录

### 站点性能

1. **图像优化**：
   - 使用适当的图像格式（WebP 或 AVIF）
   - 压缩图像大小
   - 使用响应式图像
   - 实现延迟加载

2. **资源优化**：
   - 压缩 CSS 和 JavaScript 文件
   - 使用代码分割
   - 减少 HTTP 请求
   - 使用 CDN 分发静态资源

3. **CSS 优化**：
   - 减少 CSS 文件大小
   - 使用 CSS 变量
   - 避免 CSS 阻塞渲染

4. **JavaScript 优化**：
   - 减少 JavaScript 体积
   - 延迟加载非关键 JavaScript
   - 使用现代 JavaScript 特性

5. **字体优化**：
   - 使用系统字体或优化的 Web 字体
   - 预加载关键字体
   - 减少字体变体数量

### 加载性能

1. **关键渲染路径**：
   - 优化 HTML 结构
   - 内联关键 CSS
   - 推迟非关键 JavaScript

2. **预加载**：
   - 预加载关键资源
   - 预连接到重要域名

3. **缓存策略**：
   - 为静态资源设置适当的缓存头
   - 使用内容哈希命名文件

## 安全性

### 内容安全

1. **输入验证**：
   - 验证所有用户输入
   - 防止 XSS 攻击
   - 转义特殊字符

2. **CSP (Content Security Policy)**：
   - 配置适当的 CSP 头
   - 限制脚本来源
   - 禁止内联脚本（如果可能）

3. **HTTPS**：
   - 确保站点使用 HTTPS
   - 配置 HSTS 头

### 依赖安全

1. **定期更新**：
   - 定期更新依赖项
   - 检查安全漏洞

2. **依赖分析**：
   - 使用 `cargo audit` 检查依赖安全
   - 避免使用有已知漏洞的依赖

### 部署安全

1. **环境变量**：
   - 安全管理环境变量
   - 避免在代码中硬编码敏感信息

2. **权限**：
   - 设置适当的文件权限
   - 限制服务器访问

3. **防火墙**：
   - 配置适当的防火墙规则
   - 限制不必要的端口
